# SEO 策略与项目裂变规范

## 1. 核心定位
本项目的核心目标是**精准流量获取**，而非品牌形象展示或文学创作。
- **目标受众**：中文区加密货币用户 (下载、注册、交易)。
- **语言风格**：纯正中文，杜绝翻译腔与 AI 生成痕迹。
- **关键词策略**：以 SEO 搜索词 (Query) 为绝对导向。

## 2. 关键词替换规范 (Global Replacement)
在项目裂变中，需执行严格的全局关键词替换，确保新站主题统一：
- **主词替换**：
  - `欧易` / `Ouyi` -> `OKX`
  - `欧意` / `殴易` / `OKEx` -> `OKX中文` 或 `OKX` (统一品牌认知)
- **URL (Slug) 结构**：
  - URL必须与内容主题一致，全站统一。
  - 原：`/ouyi-app` -> 新：`/okx-app`
  - 原：`/ouyi-zhuce` -> 新：`/okx-zhuce` 或 `/okx-register`

## 3. 内容创作原则 (Content Strategy)
### 3.1 拒绝"AI腔"
- **禁止**：使用“综上所述”、“随着区块链技术的发展”、“在当今数字经济时代”等无意义的铺垫。
- **禁止**：逻辑圆滑但信息密度低的废话。
- **要求**：**直接上干货**。用户搜什么，第一句话就回答什么。

### 3.2 标题与层级 (H1-H3)
标题标签是权重的核心容器，必须包含关键词：
- **H1 (页面主标题)**：**必须包含核心目标词**。
  - *Example*: "OKX App下载 - 2026最新安卓/iOS中文版官方安装包"
- **H2 (段落副标题)**：**对应长尾搜索词 (Long-tail Keywords)**。
  - *Example*: "OKX苹果手机怎么下载？(无需海外ID)"
  - *Example*: "OKX交易所安全吗？用户资金保障说明"
- **H3 (细节/步骤)**：**操作步骤或具体属性**。
  - *Example*: "第一步：注册OKX账号"
  - *Example*: "OKX手续费费率表"

### 3.3 语言纯净度
- **中文优先**：除了必要的行业术语 (BTC, ETH, USDT, Web3)，正文说明尽量使用中文。
- **避免夹杂**：不要为了洋气而使用 "Global"、"Exchange" 等词，直接用 "OKX中文版"、"交易所"。

## 4. 生产部署规范
- **项目命名**：文件夹名称必须与部署域名一致，所见即所得。
  - 原项目：`oyicex.com` (欧易主词)
  - 新项目：`xuuaph.cn` (OKX主词)
- **环境隔离**：每个域名对应独立的代码库文件夹，避免配置混淆。

## 5. SEO 内容配置化 (Content Abstraction)
为避免在业务逻辑中硬编码 SEO 文案，导致后续维护困难，必须执行 **内容与逻辑分离 (Content-Code Separation)**：
- **配置文件**：所有 SEO 相关的 H1/H2/H3、正文段落、FAQ 列表，必须提取到独立的配置文件中 (`src/content/` 或 `config/seo/`).
- **目的**：
  - 避免修改文案时误伤代码。
  - 方便快速复制出一套新的 SEO 策略 (只需替换 config 文件)。
  - 支持针对不同页面、不同关键词的批量生成。
